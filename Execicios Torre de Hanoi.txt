1.O que representa o problema da Torre de Hanói?

Resposta: Para a ciência e educação, o problema representa conceitos fundamentais:

Recursividade: A solução de mover n discos depende de saber mover n-1 discos. É o exemplo padrão usado no ensino de algoritmos de programação.
Crescimento Exponencial: O número mínimo de movimentos segue a fórmula 2^n-1, onde n é o número de discos.
Raciocínio Lógico e Planejamento: É amplamente utilizado na psicologia e psicopedagogia para avaliar a flexibilidade cognitiva, a memória de trabalho e a capacidade de resolução de problemas de um indivíduo.

2.Quem foi o criador da Torre de Hanói e em que ano ela foi proposta?

Resposta: A Torre de Hanói foi criada pelo matemático francês Édouard Lucas. O quebra-cabeça foi proposto originalmente no ano de 1883.

3.Quais são as três regras fundamentais do jogo da Torre de Hanói?

Resposta: Independentemente do que represente, o desafio deve sempre respeitar três regras:
1.Mover apenas um disco por vez.
2.Cada movimento consiste em tirar o disco do topo de uma das hastes e deslizá-lo sobre outra.
3.Nunca colocar um disco maior sobre um menor.

4.Qual é o objetivo principal do algoritmo da Torre de Hanói?

Resposta: O objetivo principal do algoritmo é transferir todos os discos da estaca de origem (geralmente a primeira) para a estaca de destino (geralmente a terceira), usando uma estaca auxiliar no processo.

5.Qual é a fórmula que calcula o número mínimo de movimentos necessários para resolver o problema com n discos?

Resposta: A fórmula para calcular o número mínimo de movimentos necessários para resolver a Torre de Hanói com n discos é 2^n-1 onde:
n: representa a quantidade de discos.
2: indica que a dificuldade dobra a cada novo disco adicionado (crescimento exponencial).
-1: subtrai o movimento final que completa a transferência.

6.Quantos movimentos são necessários para resolver o problema com 3 discos?

Resposta: Para resolver o problema com 3 discos, são necessários 7 movimentos.

7.Qual o tempo de complexidade do algoritmo da Torre de Hanói?

Resposta: O tempo de complexidade do algoritmo da Torre de Hanói é 0(2^n)

8.Por que o problema da Torre de Hanói é considerado um exemplo clássico de recursão?

resposta: Ele é o exemplo clássico porque a solução de um problema grande (mover n discos) depende obrigatoriamente da resolução de versões menores do mesmo problema (mover n-1 discos).

9.O que significa “caso base” em um algoritmo recursivo, e qual é o caso base na Torre de Hanói?

resposta: O caso base é a condição de parada de um algoritmo recursivo. Sem ele, a função chamaria a si mesma infinitamente, causando um erro de estouro de memória (o famoso StackOverflow). É o cenário mais simples possível, onde a resposta é direta e não precisa de novas divisões.

10.O que acontece com o número de movimentos totais quando se adiciona mais um disco ao problema?

Resposta: Quando você adiciona mais um disco, o número total de movimentos mais do que dobra. Matematicamente, se para discos o total é M, para n+1 discos o novo total será 2M+1.


11-Qual é o papel dos parâmetros origem, destino e auxiliar no método moverDiscos()?

Pense neles como os papéis que as estacas assumem em cada passo do jogo:
Origem: De onde o disco sai.
Destino: Para onde o disco vai.
Auxiliar: Onde o disco "espera" temporariamente para não bloquear o caminho.
Na prática, o código troca esses papéis o tempo todo para conseguir mover a torre inteira seguindo as regras.

12- O que acontece se o caso base if (n == 1) for removido do código?
Se você remover o caso base, o código entrará em uma recursão infinita.
Em termos simples:
O método continuará chamando a si mesmo para mover n-1, depois n-2, n-3... e nunca vai parar no 1.
Ele tentará mover 0 discos, depois -1 discos, -2 discos, e assim por diante.
O programa vai travar e dar um erro de Stack Overflow (estouro de memória), pois o computador esgota o espaço para guardar tantas chamadas pendentes.
O if (n == 1) é o "freio" que diz ao algoritmo: "Chegamos ao menor disco possível, agora pode parar de dividir o problema e comece a mover as peças".

13-No trecho abaixo, o que significa a linha moverDiscos(n - 1, origem, auxiliar, destino);?

moverDiscos(n - 1, origem, auxiliar, destino);
System.out.println("Mover disco " + n + " de " + origem + " para " + destino);
moverDiscos(n - 1, auxiliar, destino, origem);

Essa linha significa: "Tire a pilha de cima do caminho".
Para você conseguir mover o disco maior () da origem para o destino, você precisa primeiro mover todos os discos menores () que estão em cima dele para a estaca auxiliar.

14-Por que o algoritmo chama o próprio método dentro dele (recursão)?
O algoritmo usa a recursão porque o problema da Torre de Hanói é repetitivo por natureza: para mover uma torre de 10 discos, você precisa primeiro saber mover uma de 9; para mover a de 9, precisa saber mover a de 8, e assim por diante.


15-O que a função System.out.println() exibe em cada iteração da função recursiva?

A função exibe a instrução de movimento do disco atual.
Especificamente, ela mostra:
Qual disco está sendo movido (o número 
).
De onde ele está saindo (a estaca origem).
Para onde ele está indo (a estaca destino).
Como essa linha só é executada após a primeira chamada recursiva terminar, ela representa o movimento real da peça da base daquela sub-torre específica, documentando o passo a passo da solução no console.

16-Como o número de chamadas recursivas está relacionado ao número de discos (n)?
O número de chamadas cresce de forma exponencial. A relação é direta: cada disco a mais dobra (aproximadamente) o trabalho do algoritmo.

17-O que aconteceria se os parâmetros destino e auxiliar fossem trocados na primeira chamada recursiva?
Se você trocasse os parâmetros destino e auxiliar na primeira chamada, o algoritmo passaria a mover os discos para a estaca errada.

18-Qual é o tipo de dado utilizado para representar as hastes (A, B, C) no código?
Geralmente, utiliza-se o tipo char ou String.
Ambos servem apenas para dar um rótulo legível que o System.out.println usará para te dizer de onde o disco está saindo e para onde está indo.

19-No programa com contador de movimentos, qual é a finalidade da variável contador?
A finalidade da variável contador é registrar a quantidade total de passos executados pelo algoritmo para resolver o problema.
Geralmente, ela é incrementada (contador++) toda vez que a linha System.out.println é executada, já que cada impressão representa um disco mudando de lugar.

20-Se n = 4, quantos movimentos o programa imprimirá no total?
O programa imprimirá 15 movimentos.
Isso acontece porque, para mover os 4 discos, o algoritmo precisa mover a "torre de cima" (com 3 discos) duas vezes: uma para liberar o disco da base e outra para cobri-lo no destino final.
